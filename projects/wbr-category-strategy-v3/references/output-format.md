# 输出格式规范(v3)

> v3 在 v2 输出骨架基础上增加两点:
> 1. **`-1、上周预测复盘`** 段(由 grading.md 自动填充,LLM 不要改写)
> 2. **`4. 下周追踪`** 项必须形如 `📌 [pred_W{n}_xxx] 指标 阈值 by 周次`(供 lineage_parse 抽取)
> 3. **禁止词黑名单已废弃**,改由 `scripts/positive_lint.py` 做正向句式检查(Phase 6 critic 阶段执行)

## 二、`-1、上周预测复盘` 段的来源

由 `scripts/lineage_grade.py` 自动产出 `grading.md`,**Phase 5 drafter 直接嵌入**,不要 LLM 改写或润色。

格式示例(由 grading.md 直接复制粘贴):

```markdown
### -1、上周预测复盘

# 上周(W14)预测复盘

## 已达成 (3/5)
- ✅ pred_W14_001: 啤酒消费GTV ≥ 2.0亿
       实际 2.13 亿 vs 阈值 ≥ 2.0 亿 (Δ=+0.13)

## 偏离 (1/5)
- ❌ pred_W14_004: 白酒新客数 +5%
       实际 -8.0% vs 阈值 +5% (**需在 Phase 1 立 Q**)

## 数据未回流 (1/5)
- ⏸ pred_W14_005: 精酿 GTV ≥ 0.5亿
       Excel 中无可识别的周列(延续到下周)
```

> 该段是 v3 与 v2 最显著的视觉差异。**首次跑(无 grading.md)时,此段写"首次跑本品类 WBR,无上周报告可对照",不留空。**

## 三、`4. 下周追踪` 项的硬约束

每条 📌 必须严格符合:

```
📌 [pred_W{当前周}_{三位序号}] {指标名} {阈值} by W{未来周}
```

✅ 合格:

```
📌 [pred_W15_001] 啤酒消费GTV ≥ 2.0亿, by W16
📌 [pred_W15_002] 风险水位 < 3.0%, by W16
📌 [pred_W15_003] 啤酒新客CAC <= 50元, by W16
```

❌ 不合格(`lineage_parse.py` 会报错):

```
📌 持续关注精酿数据回流
📌 GTV 保持向好趋势
📌 关注白酒新客
📌 啤酒 GTV [缺 by 周次]
```

**阈值支持的语法**:

| 类型 | 示例 |
|------|------|
| 绝对值(>=) | `≥ 2.0亿`、`>= 50万` |
| 绝对值(<=) | `<= 50元`、`≤ 3.0%` |
| 相对百分比 | `+5%`、`-8%` |
| 相对点 | `+3pp`、`-5pp` |
| 语义(配合 prev 值) | `回升`、`止跌`、`保持` |

## 四、数值格式规范(继承 v2,无变化)

| 数据类型 | 格式规则 | 示例 |
|---------|---------|------|
| 消费 GTV | 保留 1 位小数,单位"亿" | 2.3亿 |
| 订单量 | 单位"万单" | 18.5万单 |
| 新客数 | 单位"万" | 5.2万 |
| WoW(周环比) | `WoW X%`,正数可不加+,负数加- | WoW 9%,WoW -3% |
| YoY(年同比) | `YoY+X%` / `YoY-X%` | YoY+19.3% |
| 比率/渗透率变化 | 用 **pp**,不用% | 风险水位 WoW-2.3pp |
| 达成率 | XX% | 目标达成率 91.5% |
| ROI | X.XX | ROI 2.50 |
| MTD/累计数据 | 必须标注截止日期 | 截至 W13,MTD 消费 GTV XX亿 |

## 五、写作规则(继承 v2,强化)

### 规则 1:变化必须带前后绝对值(v2 已有,v3 强约束)

| ❌ 不合格 | ✅ 合格 |
|----------|---------|
| `GTV WoW+12.4%` | `GTV 从 0.28亿 增至 0.32亿(WoW+12.4%)` |
| `新客数提升了30%` | `新客数从 4.0万 增至 5.2万(+30%)` |
| `ROI 优于均值` | `ROI 1.74,高于品类均值 1.50` |

### 规则 2:每个数字必须能 grep 回 evidence.md(v3 新增)

draft_v2.md 中出现的每个具体数字,必须能在 `runs/W{n}/evidence.md` 的某一行找到。critic 会逐条核对。

### 规则 3:逻辑链必须完整(v2 已有)

每条策略描述含「背景/动机 → 做了什么 → 效果数据」,缺一不可。

### 规则 4:单条策略描述不超过 60 字,每子品类策略段不超过 400 字(v2 已有)

### 规则 5:反例必须显式承认(v3 新增)

若 evidence.md 中某条 evidence 标 `⚠️ 部分矛盾`,draft 必须在相关段落显式承认这一矛盾,不允许悄悄忽略。

## 六、信息密度检查(v3 替代禁止词清单)

不再维护词级黑名单。由 `scripts/positive_lint.py` 在 Phase 6 critic 阶段执行:

```bash
python3 scripts/positive_lint.py draft_v1.md
```

每个独立子句必须满足以下至少一条:
1. 含数字 + 单位(`2.1亿` / `47元` / `-8pp`)
2. 含具体动作动词(`投放` / `下线` / `调整为` / `补贴`)
3. 含明确变化方向(`WoW+`、`回升`、`止跌`)

不满足 → critic 标红,revisor 重写。

> 这比 v2 的 12 词黑名单更难绕过——LLM 不能靠换词蒙混过关,**必须真的塞具体内容进去**。

---

## 七、因果声明规则(继承 v2)

- ✅ 有实验/观测证据才能写因果
- ✅ 相关性推断必须标注:`[相关性推断,待验证]`
- ❌ 禁止无证据因果
- ❌ 禁止无根据前瞻

> `[相关性推断,待验证]` 标注会被 `lineage_parse.py` 自动抽取进 `open_hypotheses`,供下周追踪。

---

## 八、归因规范(继承 v2)

| 场景 | 要求 |
|------|------|
| \|WoW\| > 10% | 必须给出驱动因素 |
| \|WoW\| > 5% | 需量化驱动因素 |
| 节日后回落 | 必须判断"正常回落"还是"异常下滑" |
| YoY 异常 | 必须检查基期可比性 |

| 因素类型 | 处理位置 |
|---------|---------|
| 节日/天气/竞对 | 「规模」段归因 |
| 补贴/运营/选品 | 「业务策略进展」段 |
