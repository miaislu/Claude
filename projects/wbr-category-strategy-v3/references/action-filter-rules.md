# 举措筛选规则（通用品类版）

---

## 一、入选必要条件

举措必须**同时满足 A + B** 才进入策略解读。

### 条件 A：有归因链（动作 → 可观测影响）

**Level 1 - 直接归因（优先呈现）**
举措直接作用于结论指标，数据已回收，有直接数字支撑。

**Level 2 - 间接归因（可呈现，需标注推理）**
举措作用于中间变量，需一步业务推理才能连到结论指标。标注推理步骤。

**Level 0 - 无归因（排除）**
效果完全未知，或效果来自多个并发举措无法区分。
例外：战略级里程碑即使无数据，可简短提及，标注"效果待跟踪"。

---

### 条件 B：异动量级达标

满足以下任一条件即可：

| 条件 | 说明 |
|------|------|
| \|WoW\| ≥ 10%（绝对量指标） | GTV、订单量等 |
| \|WoW\| ≥ 5pp（比率型指标） | 渗透率、CXR、风险水位等 |
| 战略级里程碑 | 首次跑通新模式、重要阶段完成 |
| 风险/治理指标突破红线 | 即使规模小，治理举措必须呈现 |
| 直接影响目标缺口 | 本周举措能解释为何达成/未达成目标 |

---

## 二、优先级矩阵

```
归因确定性
高  │②重要但不确定    ①核心举措（必详写）
    │                   
低  │④低优先级        ③小规模有效举措（可简写）
    └──────────────────────────────→ 影响力（GTV贡献）
       低                高
```

- **象限①**（高影响 + 高确定）→ 详写，用 BAER 框架
- **象限②**（高影响 + 不确定）→ 写，标注"[相关性推断，待验证]"
- **象限③**（低影响 + 确定）→ 简写，合并到同类策略
- **象限④**（低影响 + 不确定）→ 不写

---

## 三、与核心问题的关联（新增）

通过入选条件后，还需与 Step 0 定义的核心问题对照：

| 举措与核心问题关系 | 判断 |
|------------------|------|
| 直接回答某个核心问题 | ✅ 必写 |
| 在核心问题的关键路径上 | ✅ 要写 |
| 与核心问题无直接关系的小举措 | ⚠️ 3-5条额度内酌情 |

---

## 四、特殊类型处理

### 治理类举措
风险指标突破红线时**必须呈现**，无论GTV影响大小。

### 跨周追踪举措
- **New（首次上线）**：末尾加 📌 `下周追踪：[指标]`
- **Ongoing（延续自Wxx）**：标注 `延续自WXX，本周[新进展]`
- **Completed（已结案）**：写最终结果 + 结论

### 规划/筹备类举措
默认排除。例外：重要战略里程碑可简短提及，标注"效果待跟踪"。

---

## 五、排除清单

| 排除类型 | 示例 |
|---------|------|
| 纯会议/对接记录 | "与XX进行了一次对接" |
| 无时间节点的模糊计划 | "计划研究XX方向" |
| 上期已写、本期无新进展 | 重复内容 |
| 外部事件本身（非我方响应） | 节日/天气/竞对变化本身 |
| 效果未知且非里程碑 | 无数据、无推理链、无战略意义 |

---

## 六、数量控制

每个子品类：**3-5 条核心举措**（象限①②为主）。

> 宁可少而精，不可多而滥。
