# Evidence Table — 强制中间产物

> v3 的核心约束之一:**Phase 5 之后所有的报告文字,只允许引用证据表中的数字**。
> 这一层把"事实提取"和"叙述写作"强制分开,从根上消除"在写作时编造数字"。

---

## 一、为什么需要证据表

v2 (和大多数 AI 分析项目)的流程是 `数据 → 报告文字`,一跳完成。
LLM 在这一跳里同时背负:
- 事实提取(从 CSV 找数字)
- 叙述组织(谋篇布局)
- 取舍判断(什么进什么不进)
- 格式约束(数字精度、单位、句式)

四个任务挤在一起,LLM 最容易出错的就是**事实提取**——一句话里塞着 5 个数字时,有的来自表 A 有的来自表 B,出错没人能在事后核对。

v3 把这一跳拆成两半:
1. **Phase 3 证据表**:只做事实提取。一条 evidence 一行,标明 source。
2. **Phase 5 drafter**:只做叙述。每个数字必须能 grep 回 evidence.md。

事后人工 review 或 critic 自检都很容易:**报告里出现的数字,逐一对照证据表**。无法对照的 = 编造。

---

## 二、evidence.md 的格式

```markdown
# Evidence Table — W15 啤酒

> 强制规则:每个 Q 至少 2 条 evidence;至少 1 条 ⚠️/🔘(反例或中立)。
> 每条 evidence 必须可追溯到 source 列。

| Q | Evidence | Source | Confidence | Direction |
|---|----------|--------|-----------|-----------|
| Q1 | 白酒新客数 4.0万→3.7万 (WoW-8%) | Excel `白酒W15.xlsx` 第3行 | High | ✅ 支持 Q1 偏离 |
| Q1 | 政企渠道贡献率 35%→22% | drilldown.txt §白酒 | High | ✅ 部分解释 |
| Q1 | 白酒整体GTV WoW -2% (远小于新客降幅) | Excel `白酒W15.xlsx` 第1行 | High | ⚠️ 部分矛盾(GTV 没崩说明老客 ARPU 提升对冲) |
| Q1 | 白酒老客 ARPU 156→172 (+10.3%) | Excel `白酒W15.xlsx` 第8行 | High | ⚠️ 反例支持 老客对冲 假说 |
| Q2 | 专项券核销率 38%→41% (+3pp) | Action doc §3.2 | High | ✅ 支持 Q2 |
| Q2 | 啤酒 ARPU 156→168 (WoW+7.7%) | Excel `啤酒W15.xlsx` 第6行 | High | ✅ 支持 |
| Q2 | 持券用户数 WoW-3% | Excel `啤酒W15.xlsx` 第10行 | Med | 🔘 中立(可能持券人少了但购买力强) |
| Q3 | 精酿订单量 0.41→0.32 (WoW-22%) | Excel `精酿W15.xlsx` 第2行 | High | ✅ 异动确认 |
| Q3 | 精酿 GTV 仅 WoW-5% | Excel 第1行 | High | ⚠️ 矛盾(订单量大跌但 GTV 没崩 → 单价上涨,可能是数据口径 issue) |
```

## 三、列的语义

### 3.1 `Q` (问题归属)

每条 evidence 必须明确归属哪个 Q(Q1/Q2/Q3)。一条 evidence 可同时归属多个 Q,**用空格分隔**:`Q1 Q3`。

特殊值 `-` 表示"不直接回答任何 Q,但作为背景上下文",这种条目数量上限 3 条,**超过 3 条说明问题清单设置不全**。

### 3.2 `Evidence` (证据本身)

写法规则:

- **必须含前后绝对值**:`4.0万→3.7万` 而不是 `WoW-8%`
- **必须含 WoW 或 YoY 或 pp 标记**(若是变化量)
- **括号补充关键背景**:`(远小于新客降幅)` / `(可能是数据口径 issue)`

```
✅ "白酒新客数 4.0万→3.7万 (WoW-8%)"
✅ "专项券核销率 38%→41% (+3pp)"
❌ "白酒新客下降明显" ← 无数字
❌ "WoW-8%" ← 无前后绝对值
```

### 3.3 `Source` (来源)

必须能定位到具体文件 + 行/段:

```
✅ "Excel `白酒W15.xlsx` 第3行"
✅ "Action doc §3.2"
✅ "drilldown.txt §白酒"
✅ "anomaly.txt 第5条"
❌ "数据" ← 无定位
❌ "本周指标" ← 无定位
```

> Phase 6 critic 会逐条 grep,定位不到具体 row/段的 source 视为不合格。

### 3.4 `Confidence` (置信度)

- **High**: 数据直接读出来的事实
- **Med**: 数据是真的,但解读/外推有不确定性(如样本小、对比基数异常)
- **Low**: 推理链长、关联弱、需要外部知识印证

`Low` 条目原则上不应进入"规模与目标"段(避免给出软证据当事实),但可在策略进展或行动建议段作为辅助。

### 3.5 `Direction` (方向标记)

- `✅ 支持 Q{n}`:正向支持某个 Q 的预设方向
- `✅ 解释 Q{n}`:为某个 Q 提供归因
- `⚠️ 部分矛盾`:与某个 Q 的初步假说部分冲突 → **逼 LLM 在 draft 中承认这个矛盾**
- `🔘 中立`:与任何 Q 都不强相关,但值得记录
- `❌ 反例`:直接证伪某个 Q 的预设方向

---

## 四、强制规则

| 规则 | 触发 | 处罚 |
|------|------|------|
| 每个 Q 至少 2 条 evidence | LLM 在 Phase 3 自检 | < 2 条 → 退回重新扫数据 |
| 至少 1 条 `⚠️ 部分矛盾` 或 `🔘 中立` 或 `❌ 反例` | LLM 在 Phase 3 自检 | 全是 ✅ → 退回找反例 |
| 每条 evidence 的 source 可定位 | Phase 6 critic 检查 | 不可定位 → critique 标红 |
| draft 中的每个数字必须在 evidence 中存在 | Phase 6 critic 检查 | 不存在 → critique 标 ❌ 严重 |

第二条最关键。一份只有 ✅ 的证据表 = LLM 只找了支持自己的数据 = 不是分析,是辩护。**好分析必须自带反例**,这条机制把"自带反例"做成结构性要求。

---

## 五、什么时候允许 `🔘 中立`

如果**真的找不到反例**(本周 evidence 全部一边倒),允许用 `🔘 中立` 凑足"1 条非 ✅"的要求,但必须在 evidence 后用括号显式说明:

```
| Q1 | 持券用户数 WoW-3% | Excel | Med | 🔘 中立(降幅在自然波动范围内,可能与本周无直接关联) |
```

不允许用 `🔘 中立(无反例)` 这种偷懒写法。**找不到反例本身也是一种发现**,要解释为什么找不到。

---

## 六、evidence.md 不是公开输出

evidence.md 是**审计附件**,默认不嵌入主报告。但在 Phase 9 输出时,**路径必须列在主报告末尾的"审计附件"段**,供 review 时按需查阅。

> 把 evidence 放出来,等于把 LLM 的"草稿"放出来。这是认知诚实的最直接体现。
