# /falv qicao — 法律文件起草

> 输出规范、法条引用要求和禁止行为见 `agents/_guidelines.md`。

## 功能描述

根据用户提供的关键信息，起草符合中国法律规范的法律文件草稿。

## 用法

```
/falv qicao --type <文件类型>
/falv qicao --type 劳动合同 --party 用人单位
/falv qicao --type NDA --duration 3年
/falv qicao --type 买卖合同 --amount 50万
```

## 支持的文件类型

| 类型 | 说明 | 主要依据法规 |
|---|---|---|
| `劳动合同` | 固定期限/无固定期限/以完成一定工作任务为期限 | 劳动合同法 |
| `NDA` | 单向/双向保密协议 | 民法典、反不正当竞争法 |
| `买卖合同` | 货物买卖 | 民法典合同编 |
| `服务合同` | 技术服务/咨询服务 | 民法典合同编 |
| `租赁合同` | 房屋/设备租赁 | 民法典合同编 |
| `合作协议` | 业务合作/战略合作 | 民法典合同编 |
| `股权转让协议` | 有限公司股权转让 | 公司法 |
| `竞业限制协议` | 离职后竞业限制 | 劳动合同法第二十三条 |
| `数据处理协议` | 委托处理个人信息 | 个人信息保护法第二十一条 |

## 参数

| 参数 | 说明 |
|---|---|
| `--type <类型>` | 文件类型（必填） |
| `--party <甲/乙>` | 代表哪方起草（影响条款倾向） |
| `--amount <金额>` | 合同金额 |
| `--duration <期限>` | 合同/保密/竞业期限 |
| `--balanced` | 起草平衡版本（不偏向任何一方） |

## 执行流程

1. 收集必要信息：如关键信息缺失，先向用户提问（双方主体、核心条款等）
2. 根据类型加载对应的法定必备条款清单
3. 生成包含所有必备条款的合同草稿
4. 在草稿末尾标注"【起草说明】"，指出需要双方商定填写的空白项

## 输出格式

直接输出合同正文（Markdown 格式），以标准条款结构组织。

起草完成后询问：
- "是否需要针对此草稿运行 `/falv shencha` 进行审查？"
