# Skill: /legal onboard

## Trigger

User runs `/legal onboard`, `/legal onboard --reset`, or `/legal onboard --show`.

## Purpose

运行冷启动实践档案访谈，收集用户的法律工作背景与偏好，生成个性化配置文件并保存至 `~/.claude/legal-agent-profile.md`。后续所有法律审查任务将自动读取该文件以提供定制化服务。

---

## Flags

| Flag | 行为 |
|------|------|
| （无） | 若档案不存在则运行访谈；若已存在则提示用户使用 `--reset` |
| `--reset` | 无论档案是否存在，强制重新运行完整访谈并覆盖写入 |
| `--show` | 读取并展示当前档案内容，不运行访谈 |

---

## Step-by-Step Flow

### Step 0 — Flag Routing

- 若 flag 为 `--show`：读取 `~/.claude/legal-agent-profile.md` 并以格式化 Markdown 展示，然后退出。若文件不存在，提示"尚未建立档案，请运行 `/legal onboard` 开始配置。"
- 若 flag 为 `--reset`：跳过存在性检查，直接进入 Step 1。
- 若无 flag：检查 `~/.claude/legal-agent-profile.md` 是否存在。若存在，告知用户"档案已存在，如需重新配置请使用 `/legal onboard --reset`"并退出；若不存在，进入 Step 1。

### Step 1 — Welcome

向用户展示以下欢迎语：

```
欢迎使用法律 AI 助手配置向导。

本向导将通过 7 个问题了解您的工作背景与偏好，
生成专属实践档案，帮助助手在后续任务中提供更精准的建议。

全程约需 3 分钟，请依次作答。
```

### Step 2 — Interview Questions

逐题展示，等待用户回答后再显示下一题。每题显示题号与选项，用户可输入选项编号或直接输入自定义内容。

---

**Q1 — 使用方身份**
```
【第 1 题 / 共 7 题】您的主要身份是？

  A. 企业法务（甲方）
  B. 律师事务所律师（乙方 / 代理方）
  C. 合规官 / 风控人员
  D. 法学研究者 / 学生
  E. 其他（请说明）
```

**Q2 — 主要合同类型**
```
【第 2 题 / 共 7 题】您最常审查或起草的合同类型是？（可多选，输入编号以逗号分隔）

  1. 采购 / 供应链合同
  2. 销售 / 服务合同
  3. 技术开发 / 外包合同
  4. 保密协议（NDA）
  5. 股权投资 / 并购协议
  6. 劳动 / 用工合同
  7. 租赁 / 不动产合同
  8. 金融 / 借贷合同
  9. 其他（请说明）
```

**Q3 — 审查立场偏好**
```
【第 3 题 / 共 7 题】您通常代表哪一方进行合同审查？

  A. 固定代表甲方（需求方 / 买方）
  B. 固定代表乙方（供应方 / 卖方）
  C. 双方均有，视项目而定
  D. 中立立场（如仲裁、合规审查）
```

**Q4 — 主要行业**
```
【第 4 题 / 共 7 题】您所在或服务的主要行业是？（可多选，输入编号以逗号分隔）

  1. 科技 / 互联网
  2. 制造 / 工业
  3. 金融 / 银行 / 保险
  4. 医疗 / 生命科学
  5. 房地产 / 建筑
  6. 能源 / 环保
  7. 消费品 / 零售
  8. 教育
  9. 政府 / 国有企业
  10. 其他（请说明）
```

**Q5 — 风险偏好**
```
【第 5 题 / 共 7 题】您的合同审查风险偏好是？

  A. 保守型 — 严格保护己方利益，倾向提出较多修改意见，优先规避风险
  B. 均衡型 — 兼顾双方合理诉求，在关键条款上坚守底线
  C. 务实型 — 以促成交易为优先，仅就重大风险提出修改，接受部分不利条款
```

**Q6 — 特别关注点**
```
【第 6 题 / 共 7 题】您在合同审查中有哪些特别关注点？（可多选，输入编号以逗号分隔）

  1. 违约责任与赔偿上限
  2. 知识产权归属
  3. 数据隐私与合规（如个保法、GDPR）
  4. 争议解决条款（管辖、仲裁）
  5. 保密与竞业限制
  6. 不可抗力与免责条款
  7. 付款条件与账期
  8. 合同解除与终止
  9. 反腐败 / 合规声明
  10. 其他（请说明）
```

**Q7 — 内部审查指引（选填）**
```
【第 7 题 / 共 7 题】（选填）您的团队是否有内部合同审查指引或红线清单？

若有，请简要描述核心要点（如"赔偿上限不超过合同金额的 10%"）；
若无或暂不提供，请直接回复"无"或按 Enter 跳过。
```

---

### Step 3 — Confirm

汇总展示用户的所有回答，并询问：

```
以上信息是否准确？

  Y — 确认，生成档案
  N — 重新作答
  E — 编辑某一题（请输入题号）
```

若用户选择 N，返回 Step 2 重新开始；若选择 E，仅重新展示对应题目并更新答案，之后再次展示确认界面。

### Step 4 — Write Profile File

将档案写入 `~/.claude/legal-agent-profile.md`，格式如下：

```markdown
---
generated: <ISO 8601 日期时间，如 2026-06-04T10:30:00+08:00>
version: "1.0"
---

# 法律 AI 助手 — 实践档案

## 使用方身份与角色

<根据 Q1 答案生成一句话描述，例如："企业法务（甲方），负责合同审查与风险管控。">

## 主要业务领域

<根据 Q2 答案列出合同类型，使用无序列表>

## 审查立场偏好

<根据 Q3 答案生成描述，明确代表方及其含义>

## 行业背景

<根据 Q4 答案列出行业，使用无序列表>

## 风险偏好

**<保守型 / 均衡型 / 务实型>**

<根据 Q5 选项生成 1-2 句说明，描述该风险偏好在审查中的具体体现>

## 特别关注点清单

<根据 Q6 答案生成无序列表，每项关注点独占一行>

## 内部审查指引摘要

<若 Q7 有内容则写入；若用户跳过则写"（暂未提供）"，保留该章节供后续补充>
```

### Step 5 — Success Message

文件写入成功后，展示：

```
档案已保存至 ~/.claude/legal-agent-profile.md

后续使用 /legal review、/legal draft 等命令时，
助手将自动读取您的档案并提供个性化建议。

如需修改，请运行 /legal onboard --reset 重新配置，
或直接编辑 ~/.claude/legal-agent-profile.md 文件。
```

---

## Error Handling

- 若写入文件失败（如权限不足），提示用户手动创建 `~/.claude/` 目录后重试。
- 若用户在访谈中途中断（如输入 `quit` 或 `exit`），提示"访谈已中断，档案未保存"并退出，不写入任何文件。
- Q7 为选填题，用户回复空内容或"无"均视为跳过，档案中保留章节标题并注明"（暂未提供）"。
