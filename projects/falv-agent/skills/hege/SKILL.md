# /falv hege — 合规检查

> 输出规范、法条引用要求和禁止行为见 `agents/_guidelines.md`。

## 功能描述

针对特定法律法规对合同或文件进行合规性检查，输出合规差距报告（Gap Analysis）。

## 用法

```
/falv hege <文件路径>
/falv hege 隐私政策.txt --type pipl
/falv hege 用户协议.txt --type 电商
/falv hege 劳动合同.pdf --type 劳动法
/falv hege 数据处理协议.txt --type 数据安全
```

## 参数

| 参数 | 说明 |
|---|---|
| `--type <合规类型>` | 指定检查框架（见下方支持列表） |
| `--industry <行业>` | 指定行业（金融/医疗/教育/互联网）以启用行业特定检查项 |

## 支持的合规框架

| 类型值 | 对应法规 |
|---|---|
| `pipl` | 个人信息保护法（PIPL） |
| `数据安全` | 数据安全法 + 网络安全法 |
| `劳动法` | 劳动法 + 劳动合同法 |
| `电商` | 电子商务法 + 消费者权益保护法 |
| `公司法` | 公司法（股东协议、章程） |
| `反垄断` | 反垄断法（排他性条款检查） |
| `全面` | 以上所有框架（默认值） |

## 执行流程

1. 根据 `--type` 参数加载对应的合规检查清单
2. 调用 `he-gui-jian-cha` Agent 执行逐项检查
3. 输出差距报告：通过项、未通过项、需补充内容

## 输出格式

```markdown
## 合规检查报告

**检查框架**：个人信息保护法（PIPL）  
**合规评分**：65 / 100

---

### ✅ 已满足（X 项）
- [条款位置] 具备处理个人信息的明确告知条款

### ❌ 不合规（X 项，需立即整改）
| # | 问题 | 法律依据 | 建议补充内容 |
|---|---|---|---|
| 1 | 缺少用户撤回同意的机制 | PIPL 第十五条 | 应提供撤回同意的便捷渠道，且不影响已完成处理的合法性 |

### ⚠️ 待确认（X 项）
...

---
> ⚠️ 本报告仅供参考，不构成正式法律意见。
```
