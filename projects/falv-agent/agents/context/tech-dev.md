# 技术开发 / 软件定制合同专项上下文

适用范围：定制软件开发合同、IT 系统集成合同、技术服务合同、软件外包协议、APP 开发合同、技术咨询合同。

---

## 一、专项条款类别（补充通用分类）

| 专项类别 | 典型条款内容 |
|---|---|
| **需求与范围** | 需求文档（PRD）、功能范围、变更控制程序 |
| **交付物** | 可交付成果清单、源代码、文档、测试报告 |
| **验收标准** | 验收测试方案、通过标准、验收期限、异议处理 |
| **开发里程碑** | 阶段划分、每阶段交付物、付款节点 |
| **知识产权归属** | 定制成果归属、背景技术保留、开源组件使用 |
| **源代码交付** | 源代码是否交付、托管方式、密钥移交 |
| **保修期与缺陷修复** | 保修期限、缺陷分级、响应时限、修复义务 |
| **需求变更机制** | 变更申请流程、工作量评估、追加费用 |
| **保密义务** | 甲方业务信息、乙方技术方案的双向保密 |
| **人员稳定性** | 核心开发人员更换的限制和通知义务 |
| **第三方组件** | 开源许可证合规、商业软件授权费归属 |

## 二、风险评估专项检查清单

### 2-A 委托方（甲方）立场

#### 🔴 自动触发高危

1. **验收标准由开发方单方定义或极度模糊** ⚡
   - "达到甲方满意"、"符合行业标准"——无客观测量标准
   - 验收标准未在合同签署时确定，留待后续协商（等于无标准）
   - 验收异议处理：若无明确程序，开发方可声称"验收通过"

2. **知识产权归属不明确** ⚡
   - 未明确约定定制成果（源代码、设计稿、数据库结构）归委托方
   - 开发方保留"再利用"权利——将甲方定制功能用于其他客户
   - 委托作品 vs. 职务作品的区别：
     - **委托作品**（《著作权法》第18条）：默认归受托方（开发公司）；须合同明确约定归委托方
     - 建议：合同明确写"本合同项下全部开发成果的知识产权归甲方所有"

3. **源代码不交付或设置障碍** ⚡
   - 合同无源代码交付义务，甲方仅获得使用权（被锁定风险）
   - 源代码交付以"尾款结清"为条件但验收争议导致尾款被扣押
   - 建议：源代码在代码托管平台（如私有 GitLab）实时同步，而非交付时点

4. **需求变更无控制机制**
   - 开发方可以"范围外"为由拒绝任何调整，或无限追加费用
   - 甲方无法证明某功能在原需求范围内（需求文档不存在或不详细）

5. **开源组件许可证风险**
   - 开发方使用 GPL 协议的开源组件 → 可能要求甲方产品整体开源
   - 合同未约定"乙方保证不使用与甲方商业目的冲突的开源许可证"

#### 🟡 中等风险（委托方视角）

- 保修期过短（少于 6 个月）或保修范围仅限"重大缺陷"
- 核心开发人员可随时更换，项目交接风险由甲方承担
- 里程碑付款比例前重后轻（如：签约 50%，交付 50%），甲方筹码不足
- 文档交付未明确（无文档等于后续维护完全依赖乙方）

---

### 2-B 开发方（乙方）立场

#### 🔴 自动触发高危

1. **付款节点与验收强绑定，验收权由甲方单方行使** ⚡
   - "验收通过后付款"——甲方可无限期拖延验收
   - 验收异议处理无时限，实质上给甲方无限拖付款的权利
   - 建议：约定"甲方须在提交后 X 个工作日内完成验收；逾期视为验收通过"

2. **需求变更无追加费用机制**
   - 甲方需求持续变化但合同总价固定
   - 无书面变更确认程序，口头要求事后难以举证追费
   - 建议：任何范围变更须双方签署变更单，含工作量和费用确认

3. **知识产权全部转让但背景技术未豁免**
   - 合同要求乙方将"所有与项目相关的知识产权"转让给甲方
   - 乙方的通用技术框架、中间件、工具库被一并转让，无法在其他项目复用

4. **无限赔偿责任**
   - 软件缺陷导致甲方业务损失，乙方须全额赔偿（无责任上限）
   - 建议：责任上限 = 合同总金额；间接损失（利润损失、商誉损失）明确排除

#### 🟡 中等风险（开发方视角）

- 合同总价固定但需求文档不完整，范围蔓延风险高
- 保修期过长（超过 1 年）或保修范围包含甲方使用不当导致的问题

## 三、合规检查专项清单

### 技术合同认定（享受税收优惠的前提）

依据《技术合同认定规则》，以下情形可认定为技术合同（享受增值税减免）：

- [ ] 合同标的为技术开发、技术转让、技术咨询、技术服务四类之一
- [ ] 合同中技术性内容（技术指标、技术要求）清晰，非单纯劳务
- [ ] 合同在技术合同登记机构完成登记（享受优惠的必要前提）

### 知识产权条款合规

- [ ] 委托开发成果归属已明确约定（默认归受托方，须合同约定归委托方）
- [ ] 第三方开源组件使用已获授权，许可证与商业目的兼容
- [ ] 员工/外包人员知识产权已通过劳动合同/外包协议归属至乙方，再由乙方转让甲方

### 数据安全（开发过程中接触生产数据时）

- [ ] 开发环境不得使用真实用户数据（须脱敏处理）
- [ ] 乙方开发人员访问甲方系统的权限管理和日志记录
- [ ] 项目结束后乙方数据删除义务

## 四、关键条款建议

**验收条款标准写法：**
> "甲方应在乙方提交验收申请后 **[10] 个工作日**内完成验收测试。验收测试按《验收测试方案》（附件X）执行。甲方逾期未完成验收且未提出书面异议的，视为验收通过。"

**知识产权条款标准写法：**
> "乙方在本合同项下专门为甲方开发的全部成果（含源代码、文档、设计），其著作权及其他知识产权归甲方所有。乙方保留其在本合同签署前已拥有的背景技术（见附件Y），甲方获得在本项目范围内使用上述背景技术的永久、不可撤销许可。"
