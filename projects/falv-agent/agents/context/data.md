# 数据处理协议专项上下文

适用范围：数据处理委托协议（DPA）、数据共享协议、数据授权协议、隐私政策（审查）、个人信息跨境传输协议、数据安全协议。

---

## 一、专项条款类别（补充通用分类）

| 专项类别 | 典型条款内容 |
|---|---|
| **处理目的与范围** | 处理目的、个人信息类型、处理方式（收集/存储/使用/传输/删除）|
| **处理合法依据** | 同意、合同履行、法定义务、重大公共利益等（PIPL 第十三条）|
| **数据主体权利** | 查阅、复制、更正、删除、撤回同意、可携带权 |
| **委托处理条款** | 受托方处理权限、禁止转委托条件、审计权 |
| **安全保护措施** | 技术措施、管理措施、安全事件响应 |
| **跨境传输机制** | 标准合同条款（SCC）、安全评估、认证机制 |
| **数据本地化** | 存储地点、境外访问限制 |
| **数据留存与删除** | 留存期限、合同终止后的删除义务 |
| **违规通知义务** | 个人信息泄露的通知时限、通知对象 |
| **责任分担** | 委托方与受托方的责任边界、赔偿上限 |

## 二、风险评估专项检查清单

### 2-A 委托方立场（数据控制者）

#### 🔴 自动触发高危

1. **受托方处理权限边界不清**
   - 受托方可将数据"用于改善自身服务"、"用于统计分析"等超出委托目的的表述
   - 无明确禁止转委托或转委托需委托方书面同意的条款（PIPL 第二十一条）

2. **安全违规时委托方承担连带责任但无法控制**
   - 受托方发生数据泄露但通知义务模糊（应在 72 小时内通知委托方）
   - 无审计权或审计权需受托方配合（形同虚设）

3. **数据删除义务不明确**
   - 合同终止后受托方数据删除时限不明（建议：合同终止后 30 日内完成删除并提供证明）
   - 未约定删除确认义务（书面证明）

4. **跨境传输合规缺失**（如涉及）
   - 未明确数据存储地点和境外访问机制
   - 未约定安全评估或标准合同条款（SCC）所需的合规前提（《个人信息跨境传输标准合同办法》）

5. **责任倒置风险**
   - 合同约定受托方"免除所有由个人信息处理引发的责任"，而实际监管处罚由委托方承担

#### 🟡 中等风险（委托方视角）

- 受托方安全资质要求不明确（是否要求 ISO 27001、等保三级等）
- 个人信息泄露赔偿上限过低，无法覆盖实际损失
- 数据可携带权实现机制（用户要求数据迁移时的接口和时限）

---

### 2-B 受托方立场（数据处理者）

#### 🔴 自动触发高危

1. **责任边界不清晰，受托方承担无限连带责任**
   - 委托方违规指令导致的损失应由委托方承担，合同中须明确"受托方按委托方指令处理不承担责任，除非明知指令违法"（PIPL 第二十一条第三款）
   - 无论何种原因的数据泄露均由受托方赔偿（应区分委托方过失和受托方过失）

2. **审计权过于宽泛**
   - 委托方可随时、无预约进行现场审计，影响受托方正常运营
   - 审计范围无边界（应限于与本合同数据处理相关的系统和记录）

3. **处理目的限制影响受托方合法业务**
   - 禁止受托方使用匿名化/去标识化后的统计数据（即便用于服务改善）

#### 🟡 中等风险（受托方视角）

- 删除义务时限过短，技术上难以实现
- 安全措施标准过高且随委托方单方要求调整（未固定在协议中）

## 三、合规检查专项清单

### PIPL 委托处理必查项（《个人信息保护法》第二十一条）

- [ ] 明确委托处理的目的、期限、方式、个人信息的种类、保护措施及双方权利义务
- [ ] 受托方不得超出约定的处理目的、处理方式等处理个人信息
- [ ] 委托合同不生效、无效、被撤销或终止后，受托方须将个人信息返还或删除
- [ ] 未经委托方同意，受托方不得转委托

### 数据安全法合规必查项

- [ ] 是否涉及重要数据（须建立数据安全管理制度）
- [ ] 数据分类分级是否明确
- [ ] 跨境传输重要数据的安全评估要求

### 跨境传输合规必查项（如涉及）

依据《个人信息保护法》第三十八至四十条及《个人信息跨境传输标准合同办法》：

| 情形 | 合规路径 |
|---|---|
| 关键信息基础设施运营者 / 处理超 100 万人个人信息 | 须通过国家网信办安全评估 |
| 通过专业机构认证 | 按国家网信办认证规范执行 |
| 一般情形 | 与境外接收方签订标准合同（SCC），并向网信办备案 |

- [ ] 已确定适用的跨境传输合规路径
- [ ] 标准合同中已约定境外接收方承担的义务
- [ ] 已进行个人信息保护影响评估（PIA）
