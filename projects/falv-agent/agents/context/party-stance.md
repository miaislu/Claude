# 立场框架（Party Stance Framework）

本文件定义不同立场如何影响审查角度和严格程度。所有 Agent 在收到 session_context 后，按此框架调整分析行为。

---

## 一、立场对分析行为的影响

### 1.1 单方委托模式（委托方立场明确）

```
对【委托方】不利的条款：
  → 严格标注，宁可误报，不可漏报
  → 风险评分上调 1–2 分（更保守）
  → 修改建议优先生成
  → 在输出中用 ⚡ 标记"直接影响你的条款"

对【对方】不利的条款：
  → 正常识别，说明但不强调
  → 不主动建议对方修改（除非存在法律无效风险）
```

### 1.2 平衡分析模式

```
双方条款使用同等严格程度
标注"对甲方不利"/"对乙方不利"以示区分
不偏向任何一方的修改建议
适用于：律师尽调、第三方审查、合同谈判前的全貌梳理
```

## 二、各合同类型的立场映射

### 投资协议

| 立场 | 关注重点 | 严格程度 |
|---|---|---|
| **投资方** | 优先权保护、对赌可执行性、回购触发条件、反稀释强度、信息权完整性、创始人锁定 | 对任何削弱投资方权益的条款严格标注 |
| **创始人（被投方）** | 控制权稀释、经营自主权限制、过度同意权清单、对赌指标合理性、稀释保护是否公平、退出限制 | 对任何限制创始人经营自由或过度摊薄的条款严格标注 |
| **平衡** | 全面梳理，标注利弊各方 | 均等 |

### 劳动合同

| 立场 | 关注重点 | 严格程度 |
|---|---|---|
| **用人单位** | IP 归属完整性、竞业条款可执行性、试用期合规、工资总成本、解除条件 | 对暴露用人单位法律风险的条款严格标注 |
| **劳动者** | 违法条款识别、工资支付保障、解除补偿权利、竞业补偿是否约定、试用期是否超限 | 对损害劳动者权益的条款严格标注；未约定但法定应有的权利主动提示 |
| **平衡** | 均等 | 均等 |

### 买卖合同 / 服务合同

| 立场 | 关注重点 | 严格程度 |
|---|---|---|
| **买方（甲方）** | 交付标准不明确、付款条件被动、验收权利、违约救济 | 对付款风险和收货风险严格标注 |
| **卖方（乙方）** | 付款时限保障、验收标准单方设定、责任兜底条款、知识产权保留 | 对账期风险和无限赔偿风险严格标注 |
| **平衡** | 均等 | 均等 |

## 三、输出中的立场标识

在报告中统一使用以下标识：

| 标识 | 含义 |
|---|---|
| ⚡ | 直接影响委托方的条款（单方委托模式专用） |
| 🔵 | 对甲方/投资方有利 |
| 🔴 | 对甲方/投资方不利（高风险） |
| 🟡 | 对乙方/创始人不利（中等风险） |
| ⚖️ | 平衡条款，双方责任对等 |

---

## 四、严格程度对评分的具体影响

在单方委托模式下，风险评估师对"委托方不利条款"的评分调整规则：

```
原始评分 4分（低风险）+ 单方委托调整 → 最终标注为 5–6分（中等风险）
原始评分 6分（中等风险）+ 单方委托调整 → 最终标注为 7–8分（高危）
原始评分 8分（高危）   → 保持或上调至 9分，加 ⚡ 标记
```

核心原则：**从委托方利益最大化保护的角度，宁严勿松**。
