# 立场框架（Party Stance Framework）

本文件定义不同立场如何影响审查角度和严格程度。所有 Agent 在收到 session_context 后，按此框架调整分析行为。

---

## 零、风险归属判断（必须在评分前执行）

**这是最容易出错、最重要的一步。**

在对任何条款打风险分之前，必须先判断：**这个条款的风险由谁承担？**

```
STEP 1: 识别条款受益方

  这个条款主要保护哪一方？
  
  → 如果主要保护【委托方】（即条款对委托方有利）：
  
      评估的是"对方能否成功挑战/逃脱这个条款"
      
      仅当满足以下两个条件时，才构成委托方风险：
        条件A：对方挑战成功的概率 ≥ 中等
        条件B：挑战成功后委托方蒙受的损失 ≥ 重大
      
      两个条件都满足 → 一般风险
      两者均高          → 重大风险
      
      ⚠️ 不得因"对方有权利挑战"就将委托方保护性条款标为高风险
      ⚠️ 不得因"条款措辞对对方不公平"就认为委托方有风险
      
  → 如果主要保护【对方】（即条款对委托方不利）：
  
      直接评估委托方的损失敞口（概率 × 数额/影响程度）
      → 这才是委托方真正应关注的风险

STEP 2: 一张表帮助判断

  ┌─────────────────────────────┬──────────────────────────────┐
  │ 条款类型                     │ 对收购方/委托方的处理          │
  ├─────────────────────────────┼──────────────────────────────┤
  │ 保护委托方的条款（如对方违约  │ 评估"对方能否逃脱"的概率       │
  │ 时委托方有权扣押款项）        │ 概率低 → 轻微/不列入          │
  │                             │ 概率高+损失大 → 一般/重大      │
  ├─────────────────────────────┼──────────────────────────────┤
  │ 赋予对方权利的条款（如对方有  │ 直接评估对委托方的影响          │
  │ 权请求付款、终止合同等）      │ 影响大 → 重大/一般             │
  ├─────────────────────────────┼──────────────────────────────┤
  │ 对委托方有利但执行力存疑的条  │ 评估执行力（法条依据是否充分、  │
  │ 款（如范围过宽的竞业限制）    │ 司法倾向如何）                 │
  │                             │ 执行力弱 → 一般                │
  └─────────────────────────────┴──────────────────────────────┘

STEP 3: 常见误判模式（必须避免）

  ✗ 错误：收购方设计了对卖方不公平的条款 → 标注为"收购方重大风险"
    正确：这是收购方的权利，不是收购方的风险
    （除非不公平程度达到显失公平，卖方有较大概率成功撤销）
    
  ✗ 错误："该条款措辞对委托方有利，但如果对方提出仲裁就有风险" 
    → 标注为重大风险
    正确：对方提出仲裁的权利不等于委托方有风险
    只有"对方胜诉概率高+委托方损失大"才是真正风险
    
  ✗ 错误：条款语言对对方模糊或不利 → 认为委托方面临仲裁风险
    正确：模糊条款对委托方有利（解释空间）
    仅当模糊条款对委托方不利时，才标注为委托方风险
```

## 一、立场对分析行为的影响

### 1.1 单方委托模式（委托方立场明确）

```
对【委托方】不利的条款（加重义务 / 赋予对方权利）：
  → 严格标注，宁可误报，不可漏报
  → 风险评分基于损失敞口评估（概率 × 数额/影响程度）
  → 修改建议优先生成

对【委托方】有利的条款（保护委托方 / 对对方不利）：
  → 评估"可执行性"而非"风险"
  → 若条款对委托方保护完整且执行力强 → 列入"优势条款"，不标风险
  → 若条款执行力存疑（范围过宽/无法律依据/有被撤销风险）→ 标注为"保护性条款存在执行风险"
  → 风险程度基于"保护失效的概率 × 保护失效后委托方的损失"
```

### 1.2 平衡分析模式

```
双方条款使用同等严格程度
标注"对甲方不利"/"对乙方不利"以示区分
不偏向任何一方的修改建议
适用于：律师尽调、第三方审查、合同谈判前的全貌梳理
```

## 二、各合同类型的立场映射

### 投资协议

| 立场 | 关注重点 | 严格程度 |
|---|---|---|
| **投资方** | 优先权保护、对赌可执行性、回购触发条件、反稀释强度、信息权完整性、创始人锁定 | 对任何削弱投资方权益的条款严格标注 |
| **创始人（被投方）** | 控制权稀释、经营自主权限制、过度同意权清单、对赌指标合理性、稀释保护是否公平、退出限制 | 对任何限制创始人经营自由或过度摊薄的条款严格标注 |
| **平衡** | 全面梳理，标注利弊各方 | 均等 |

### 劳动合同

| 立场 | 关注重点 | 严格程度 |
|---|---|---|
| **用人单位** | IP 归属完整性、竞业条款可执行性、试用期合规、工资总成本、解除条件 | 对暴露用人单位法律风险的条款严格标注 |
| **劳动者** | 违法条款识别、工资支付保障、解除补偿权利、竞业补偿是否约定、试用期是否超限 | 对损害劳动者权益的条款严格标注；未约定但法定应有的权利主动提示 |
| **平衡** | 均等 | 均等 |

### 买卖合同 / 服务合同

| 立场 | 关注重点 | 严格程度 |
|---|---|---|
| **买方（甲方）** | 交付标准不明确、付款条件被动、验收权利、违约救济 | 对付款风险和收货风险严格标注 |
| **卖方（乙方）** | 付款时限保障、验收标准单方设定、责任兜底条款、知识产权保留 | 对账期风险和无限赔偿风险严格标注 |
| **平衡** | 均等 | 均等 |

## 三、输出中的立场标识

在报告中统一使用以下标识：

| 标识 | 含义 |
|---|---|
| ⚡ | 直接影响委托方的条款（单方委托模式专用） |
| 🔵 | 对甲方/投资方有利 |
| 🔴 | 对甲方/投资方不利（高风险） |
| 🟡 | 对乙方/创始人不利（中等风险） |
| ⚖️ | 平衡条款，双方责任对等 |

## 四、严格程度对评分的具体影响

在单方委托模式下，风险评估师对"委托方不利条款"的评分调整规则：

```
原始评分 4分（低风险）+ 单方委托调整 → 最终标注为 5–6分（中等风险）
原始评分 6分（中等风险）+ 单方委托调整 → 最终标注为 7–8分（高危）
原始评分 8分（高危）   → 保持或上调至 9分，加 ⚡ 标记
```

核心原则：**从委托方利益最大化保护的角度，宁严勿松**。
