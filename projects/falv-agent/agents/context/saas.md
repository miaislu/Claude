# SaaS / 云服务订阅协议专项上下文

适用范围：SaaS 软件订阅服务协议、云计算服务协议（IaaS/PaaS/SaaS）、企业软件订阅合同、API 服务协议、平台即服务协议。

> 与 `data.md`（数据处理协议）的区别：SaaS 合同的核心风险是**服务可用性、数据主权和锁定风险**；数据处理协议的核心是**个人信息合规**。两者可能同时适用。

---

## 一、专项条款类别（补充通用分类）

| 专项类别 | 典型条款内容 |
|---|---|
| **服务级别协议（SLA）** | 可用性承诺（99.9% 等）、计算方式、赔偿机制 |
| **服务范围** | 功能模块、用户数/席位/使用量限制 |
| **订阅计费** | 订阅周期、自动续费、价格调整机制 |
| **数据归属与导出** | 客户数据所有权、导出格式、导出窗口期 |
| **服务终止后数据处理** | 合同终止/到期后数据留存期、删除方式 |
| **功能变更权利** | 服务商单方修改/下线功能的权利 |
| **停止服务通知** | 服务下线前的提前通知期 |
| **安全与合规** | 安全认证、数据本地化、等级保护 |
| **子处理商** | 使用第三方基础设施的告知义务 |
| **使用限制** | 禁止行为、竞争性使用限制 |

## 二、风险评估专项检查清单

### 2-A 企业用户（客户）立场

#### 🔴 自动触发高危

1. **SLA 赔偿上限远低于实际损失** ⚡
   - 常见陷阱："服务不可用超过 X 小时，赔偿当月费用的 10%"
   - 实际影响：服务中断 1 天 → 月费的 1/30 ≈ 3.3%，赔偿额极低
   - 赔偿上限仅为月费或年费的某个百分比，无法覆盖业务损失
   - 建议：争取赔偿不设上限（或上限提高），并加入 SLA 未达标触发合同解除权

2. **数据无法导出或锁定风险** ⚡
   - 合同终止/到期后：数据删除期短于数据导出所需时间（如：30 天删除但导出需要更长时间）
   - 数据导出格式为专有格式，无法迁移到其他系统
   - 服务商破产/停止运营时数据处置无预案条款
   - 建议：明确数据以通用格式（CSV/JSON/XML）导出权，且至少有 **90 天**数据保留期

3. **服务商可单方下线功能** ⚡
   - 服务商可随时调整、合并、下线功能模块，无须补偿
   - 无"重大功能变更"的预通知要求（建议：重大变更须提前 60 天通知）
   - 用户已基于某功能做了系统集成，功能下线后集成失效

4. **自动续费陷阱** ⚡
   - 自动续费条款隐藏在服务条款深处，取消须提前 X 天通知否则自动续费
   - 年付合同自动续费后不退款
   - 建议：自动续费须在续费日前 **30 天**以邮件显著提示；用户有权在续费前取消

5. **价格单方调整权**
   - 服务商有权在通知后（通常 30 天）单方调整订阅价格
   - 用户若不接受涨价，唯一选择是终止合同（但仍须支付剩余订阅期费用）
   - 建议：约定价格锁定期（至少 1 年）；涨价超过一定幅度（如 10%）赋予用户解除权

6. **停止服务无充分通知期**
   - 服务商可提前 30 天通知停止服务（企业迁移数据和替代系统部署远不止 30 天）
   - 建议：服务停止须提前至少 **180 天**通知

#### 🟡 中等风险（用户视角）

- 可用性计算中排除计划维护时间（每月计划维护可能占用大量时间）
- 用户数/席位超出须立即付费但难以实时追踪
- API 调用量限制触发后自动按超量计费（无预警机制）
- 安全事件通知时限不明确（建议：72 小时内通知）

---

### 2-B SaaS 服务商立场

#### 🔴 自动触发高危

1. **SLA 承诺过高且无豁免条款** ⚡
   - 承诺 99.99% 可用性（年停机 < 52 分钟）但基础设施无法保证
   - 无不可抗力、第三方基础设施故障、DDoS 攻击等豁免情形
   - 赔偿无上限，大客户服务中断可能面临巨额索赔

2. **数据安全责任过重**
   - 服务商承担数据泄露的全部责任，即使是用户账号泄露（用户侧原因）导致
   - 建议：明确责任边界（服务商负责平台侧安全，用户负责账号和访问安全）

3. **用户恶意使用导致服务商连带责任**
   - 用户用 SaaS 平台处理非法内容，合同未约定用户须承担全部责任
   - 建议：禁止使用条款须具体，并约定违规使用由用户承担全部法律责任

#### 🟡 中等风险（服务商视角）

- 知识产权条款：用户通过 SaaS 平台上传的内容如侵权，服务商的责任豁免须明确
- 竞争性使用：用户将服务用于与服务商竞争的产品（如逆向工程、抓取数据）

## 三、合规检查专项清单

### 数据安全与个人信息保护

- [ ] 涉及个人信息处理的，须签署数据处理协议（DPA）（参考 `data.md`）
- [ ] 数据存储地点符合本地化要求（金融、医疗等行业有强制要求）
- [ ] 等保合规：关键信息系统须通过等保三级认证
- [ ] 数据安全事件通知时限：须在发现后 72 小时内通知用户

### 云服务合规（中国特定要求）

- [ ] 境内提供云服务的外资服务商须满足等保要求
- [ ] 跨境数据传输须符合 PIPL 跨境传输规定（若数据从中国传至境外处理）
- [ ] 金融行业：须使用经银保监会认可的金融云服务商

### SLA 关键指标参考

| 可用性等级 | 年停机时间 | 月停机时间 |
|---|---|---|
| 99% | 87.6 小时 | 7.3 小时 |
| 99.9% | 8.76 小时 | 43.8 分钟 |
| 99.95% | 4.38 小时 | 21.9 分钟 |
| 99.99% | 52.6 分钟 | 4.4 分钟 |

> 企业核心业务系统建议要求至少 **99.9%** 可用性，并核实服务商基础设施是否支撑该承诺。
