# 触发测试规范 — contract-review

## Should Trigger（应触发预审流程）

| # | 用户输入 | 触发原因 |
|---|---------|---------|
| 1 | 消息包含 `file.vip.neixin.cn/...contract.docx` 链接 | 大象文件链接 |
| 2 | 消息前缀含「当前内容包含文件链接」 | 系统提示注入 |
| 3 | 消息包含 `https://xxx.com/contract.pdf` 直接链接 | .pdf 下载链接 |
| 4 | "帮我审查一下这份合同"（+文件链接） | 明确审查意图 + 文件 |
| 5 | "这份合同有没有风险"（+文件链接） | 风险检查意图 + 文件 |
| 6 | "合同预审"（+文件链接） | 触发词命中 |
| 7 | "帮我看看这份合同"（+文件链接）且无查阅意图词 | 触发词命中，无豁免词 |

## Should NOT Trigger（不应触发，或需询问后才触发）

| # | 用户输入 | 原因 |
|---|---------|------|
| 1 | "把刚才那份合同**发我**看看"（含文件链接） | 豁免词"发我"，需先询问是否预审 |
| 2 | "这份合同**下载**下来"（含文件链接） | 豁免词"下载" |
| 3 | "**转发**这个合同给他"（含文件链接） | 豁免词"转发" |
| 4 | "合同怎么发起" | 无文件，应走 contract-query-skill |
| 5 | "帮我查合同 CO260312000224" | 查询意图，应走 contract-search |
| 6 | "合同归档流程是什么" | 问询类，应走 contract-query-skill |

## 触发豁免词完整列表

用户消息同时满足触发条件 + 包含以下任意词时，询问"是否需要预审"：

`看看` / `发我` / `参考` / `下载` / `备份` / `转发` / `分享`

## 功能测试用例

### 用例 1：我方模板 + 有清单 + 有风险

```
Given: 用户上传我方模板合同（可被暗码识别）
When: 完整执行预审流程
Then:
  - 步骤③：展示识别到的模板名称，等待确认
  - 步骤④：展示通用清单 + 自定义清单列表
  - 步骤⑦：输出包含红线数量的风险报告 + 链接（合并一条消息）
  - session 文件最终 step="completed"
```

### 用例 2：对方模板 + 仅通用清单 + 无风险

```
Given: 用户上传对方模板合同（暗码识别失败或无 templateCode）
When: 完整执行预审流程
Then:
  - 步骤③：告知已标记对方模板，等待确认
  - 步骤④：用户选择仅通用清单
  - 步骤⑦：输出"未发现风险" + 链接（合并一条消息）
```

### 用例 3：对话中断后恢复

```
Given: 流程执行到步骤④（清单选择），用户关闭会话
When: 用户重新上传同一文件
Then:
  - Skill 检测到未完成的 session
  - 提示用户"发现未完成的预审任务，是否继续？"
  - 用户选择继续后，从步骤④重新开始
```
