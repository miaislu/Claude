# 交互话术模板 — 合同预审

## 整体交互原则

1. **能自动的不问用户**：文件上传、解析、模板识别全部静默完成，完成后一次性告知
2. **用户确认节点只有 3 个**：① 合同来源确认 ② 审查清单选择 ③ 补充审查事项（可跳过）
3. **失败降级静默处理**：模板识别失败/超时不报错，直接标记对方模板继续
4. **进度简短告知**：每个等待阶段给一句状态提示，不沉默
5. **用户意图模糊时主动引导**：不要让用户猜下一步该说什么

---

## 步骤②：文件接收 & 自动处理中

> 上传成功后立即回复，告知正在处理，减少用户等待焦虑

```
已收到合同文件「{fileName}」，正在解析并识别模板，请稍候…
```

---

## 步骤③：合同来源确认 ⏸️

> 模板识别完成后展示结果，**两种情况都要等用户确认或更正**

---

### 情况 A：识别为我方模板（templateCode 非空）

> identifyCode / identifyResult 字段忽略不处理

```
✅ 已识别为我方模板

模板名称：{templateName}

模板编号：{templateCode}-V{templateVersion}

合同来源是否正确？

1. 正确，继续预审
2. 识别有误，重新指定模板
3. 非我方模板，按对方模板继续
```

**用户回复「1」/「正确」/「是」/「确认」类：**
→ 保存 templateCode + templateVersion，标记 `OUR_TEMPLATE`，进入步骤④

**用户回复「2」/「识别有误」/「换一个」/「重新指定」类：**
→ 进入**情况 C：我方模板搜索**

**用户回复「3」/「对方模板」/「不是我方」类：**
→ 清空 templateCode，标记 `OPPOSITE_TEMPLATE`，直接进入步骤④（无需再次确认）

> ⚠️ 大象消息格式说明：
> - 模板名称、编号各自独立一行，字段前加「模板名称：」「模板编号：」标签
> - 编号和版本号合并：`{templateCode}-V{templateVersion}`
> - 段落之间用空行（`\n\n`）分隔，单个 `\n` 在大象里不可靠
> - 选项用数字序号，不用「·」符号

---

### 情况 B：识别为对方模板 / 识别失败

```
📄 未识别到我方模板标识，已标记为对方模板

合同来源是否正确？

1. 正确，继续预审
2. 有误，指定我方模板
```

**用户回复「1」/「正确」/「是」/「确认」类：**
→ 保持 `OPPOSITE_TEMPLATE`，进入步骤④

**用户回复「2」/「有误」/「我方模板」/「指定模板」类：**
→ 进入**情况 C：我方模板搜索**

---

### 情况 C：用户更正为我方模板（需搜索模板库）

> 调用 `listUserPermittedTemplateAttachments` 接口，见 template_search_spec.md

**第一步：引导用户提供信息**

```
好的，帮你按我方模板处理。

请提供以下任意一项，以便找到正确的模板：
  · 模板编号（如 TP240422000446）
  · 模板名称关键词（如「网络代售合作协议」）
  · 合同模板链接（contract.sankuai.com/...）

如不确定，回复「搜索全部」查看你有权限的模板列表（展示前 10 条）。
```

**第二步：根据用户输入搜索**

| 用户输入 | 解析方式 | 搜索字段 |
|---------|---------|---------|
| `TP` 开头字符串 | 识别为模板编号 | `templateCode` 精确搜索 |
| `contract.sankuai.com/...` 链接 | 解析链接中的模板编号或 ID | `templateCode` 或 `referenceId` |
| 其他文字 | 识别为模板名称关键词 | `templateName` 模糊搜索 |
| 「搜索全部」 | 不带关键词，返回全部有权限模板 | 不传搜索字段，按更新时间倒序 |

**第三步：展示搜索结果**

找到 **0 条**：
```
未找到匹配的模板，可能是模板名称/编号有误，或你暂无该模板的使用权限。

请重新输入，或回复「按对方模板继续」跳过此步骤。
```

找到 **1 条**：
```
找到以下模板，请确认：

📄 {attachmentName}
编号：{templateCode}（版本 {templateVersion}）

回复「确认」使用此模板，或回复「不对」重新搜索。
```

找到 **多条**（最多展示 5 条，超出提示缩小范围）：
```
找到 {n} 个匹配模板，请选择：

1. {attachmentName1}（{templateCode1}，版本 {templateVersion1}）
2. {attachmentName2}（{templateCode2}，版本 {templateVersion2}）
3. {attachmentName3}（{templateCode3}，版本 {templateVersion3}）
···

回复序号选择，或回复「不对」重新搜索。
```

超过 5 条时额外提示：
```
（共找到 {totalCount} 条，建议补充更精确的名称或编号缩小范围）
```

**第四步：用户选定后确认**

```
✅ 已选择我方模板：

📄 {attachmentName}
编号：{templateCode}（版本 {templateVersion}）
```

→ 保存 templateCode + templateVersion，标记 `OUR_TEMPLATE`，进入步骤④

---

## 步骤④：选择审查清单 ⏸️

> 调用 queryRules 接口获取最新清单列表，通用清单必选不可取消

**有自定义清单时：**

> ⚠️ **必须展示全部自定义清单**，不得截断或只展示部分。
> 调用接口时传 `pageSize=50`，确保一次性拿完所有清单（当前共 29 个）。
> 用 `data.page.totalCount` 校验：若 pageList 数量 < totalCount，需分页继续拉取直到拿完全部。

```
请选择本次预审的审查清单：

✅ 通用审查清单（必审，已默认选中）

可叠加以下自定义清单（共 {total} 个）：
  1. {自定义清单1名称}
  2. {自定义清单2名称}
  3. {自定义清单3名称}
  ···（全部列出，不截断）
  {total}. {自定义清单N名称}

> ⚠️ **序号规则**：此处序号为展示用的顺序编号（1、2、3…），与清单的数据库 id 无关。用户回复序号后，按序号对应到对应清单的 `id` 字段，再拼入 `checklist` 入参。

回复序号叠加选择（可多选，如「1」或「1,3,5」），或直接回复「确认」仅使用通用清单。
```

**仅有通用清单时（无自定义清单）：**

```
将使用通用审查清单（{n} 条规则）进行审查。

请回复「确认」继续，或告知需要重点关注的事项。
```

**用户选择处理：**
- 回复序号（如「1」「1,2」「1 2」） → 叠加对应自定义清单，汇总后展示已选清单再确认
- 回复「确认」/「通用就行」/「直接审」类 → 仅使用通用清单，进入步骤⑤

---

## 步骤⑤：补充审查事项 ⏸️（可跳过）

```
是否有需要重点关注的审查事项？（选填，最多 500 字）

例如：请重点关注违约金是否对等、知识产权归属是否约定明确……

如无，回复「跳过」或「没有」直接开始审查。
```

**用户回复「跳过」/「没有」/「没」/「无」类：**
→ additionalNotes 传空字符串，进入步骤⑥

**用户输入具体内容：**
→ 保存为 additionalNotes，进入步骤⑥

## 步骤⑥：提交中状态提示

```
⏳ 正在提交预审，请稍候…
```

提交成功，轮询等待结果中（每 60 秒发一次，最多 2 次）：

```
⏳ 预审进行中，通常需要 1～3 分钟，完成后会立即通知你…
```

## 步骤⑦：输出审查结果

> 合并为一条消息发出（含摘要 + 链接），禁止分两条消息发送。
> 输出格式以 output_spec_pre_review.md 为准，以下格式仅供参考。

**摘要报告（含链接，合并一条）**

```
✅ 合同预审完成！

📄 合同：{fileName}
🏷️ 合同来源：{我方模板「{templateName}（{templateCode}）」 / 对方模板}

{若有红线风险：}
🔴 红线风险（{redCount} 项）
· {auditItem}：{riskDescript 简述，不超过 40 字}
· {auditItem}：{riskDescript 简述}
···

{若有一般风险：}
🟡 一般风险（{generalCount} 项）
· {auditItem}：{riskDescript 简述}
···

{若无任何风险：}
🎉 未发现风险，合同整体符合审查要求。
```

在摘要末尾追加链接（同一条消息，不另发）：

```
📎 完整审查报告（含逐条原文定位）：
https://contract.sankuai.com/pre-review-detail?auditBillNumber={auditBillNumber}&auditBillVersion={auditBillVersion}
```

## 异常 & 错误提示

### 文件格式不支持

```
❌ 暂不支持该文件格式，请上传 .docx 或 .pdf 文件。
```

### 文件上传失败

```
❌ 文件上传失败，请检查网络后重新上传。
```

### 模板搜索无权限 / 接口异常

```
⚠️ 模板库暂时无法访问，已按对方模板继续处理。
如需指定我方模板，请稍后重试。
```

（静默降级为对方模板，不中断流程）

### 预审提交失败

```
❌ 预审提交失败，请稍后重试。
如问题持续，可前往系统手动发起：
https://contract.sankuai.com/pre-review-list
```

### 审查超时（轮询超过 6 分钟）

```
⚠️ 审查任务仍在处理中，超出等待时间，请稍后前往系统查看结果：
https://contract.sankuai.com/pre-review-list
```

### 审查失败（auditBillStatus = 20）

```
❌ 预审任务执行失败，请稍后重试，或前往系统查看详情：
https://contract.sankuai.com/pre-review-list
```
